# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in the BigBang Auth Core, please report it responsibly.

✅ **Do NOT** open a public GitHub issue for security-sensitive bugs.

### How to Report

- Email: bigbangcoindex@gmail.com  _(placeholder — can be updated)_
- Include:
  - Detailed description of the issue
  - Impact assessment
  - Steps to reproduce (PoC if possible)

## Response Policy

- We aim to acknowledge valid reports within **72 hours**
- Fixes will be released as soon as reasonably possible
- Reporters may be credited after remediation (optional)

## Scope

- Smart contracts in this repository
- Authentication logic and OTP mechanisms
- Manager registration and fee handling

Thank you for helping keep BigBang secure.
